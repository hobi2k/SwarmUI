# 2026-04-12 업데이트 기록

## 이번 수정 목표

1. SwarmUI 재시작 시 ComfyUI 대기열 패널을 자동 초기화한다.
2. 갤러리에 삭제 기능을 추가한다.
3. 갤러리에 정렬 순서 변경 기능을 추가한다.
4. 업데이트 체크 시 `DLNodes` 폴더 미존재로 나는 오류를 수정한다.
5. Ctrl+C로 서버를 종료할 때 프로세스가 남아있는 문제를 수정한다.

---

## 코드 변경

### 1. 재시작 시 ComfyUI 대기열 자동 초기화

수정 파일:

- `src/BuiltinExtensions/ComfyUIBackend/ComfyUIAPIAbstractBackend.cs`

적용 내용:

- `HasDoneStartupClear` 플래그를 추가했다.
- 백엔드가 처음 `RUNNING` 상태가 될 때 ComfyUI의 `/history`와 `/queue`에 `clear: true` POST 요청을 보낸다.
- 같은 실행 중 재연결 시에는 `HasDoneStartupClear = true`이므로 두 번 초기화되지 않는다.

배경:

- `OwnedPromptIds`는 인메모리 데이터라 재시작 시 초기화된다.
- 이 상태에서 ComfyUI에 이전 작업이 남아있으면 삭제 요청이 차단되고 대기열 패널에 이전 항목이 그대로 보인다.
- 시작 시 ComfyUI history/queue를 clear해서 이 불일치를 해소한다.
- 갤러리(LiteDB)는 이 초기화와 무관하게 유지된다.

### 2. 갤러리 삭제 기능

수정 파일:

- `src/Accounts/UserOutputHistoryIndex.cs`
- `src/WebAPI/T2IAPI.cs`
- `src/wwwroot/js/genpage/gentab/outputhistory.js`

적용 내용:

- `UserOutputHistoryIndex.DeleteEntry(user, entryId)` 메서드를 추가했다.
  - LiteDB에서 해당 항목을 제거한다.
  - 실제 파일과 사이드카 파일(`.txt`, `.swarm.json` 등)도 함께 삭제한다.
  - `RecycleDeletedImages` 설정을 따른다.
- `T2IAPI.DeleteIndexedImage` API 엔드포인트를 추가했다 (`/API/DeleteIndexedImage`).
  - `entry_id`를 받아 `DeleteEntry`를 호출한다.
  - 소유권 검증은 `DeleteEntry` 내부에서 처리한다.
- `buttonsForImage()`에 `entryId` 파라미터를 추가했다.
  - `isIndexed` 이미지이고 `entryId`가 있을 때만 Delete 버튼을 표시한다.
  - 삭제 후 갤러리 DOM에서 해당 카드를 제거한다.
- `buildOutputFileDescription()`에서 `entry_id`를 `buttonsForImage`에 전달한다.

### 3. 갤러리 정렬 컨트롤

수정 파일:

- `src/wwwroot/js/genpage/gentab/outputhistory.js`

적용 내용:

- `imageGalleryBrowser` 생성 시 `extraHeader`에 정렬 셀렉트(날짜/이름)와 역순 체크박스를 추가했다.
- `listOutputGalleryFolderAndFiles`가 DOM 요소 또는 `localStorage`에서 정렬 설정을 읽어 `ListIndexedImages` API에 전달한다.
- 정렬 설정은 `localStorage`에 저장되어 새로고침 후에도 유지된다.
- 이전에 날짜 기준 고정 + `mapped.reverse()`로 처리하던 방식을 API 파라미터 기반으로 교체했다.

### 4. DLNodes 폴더 미존재 오류 수정

수정 파일:

- `src/BuiltinExtensions/ComfyUIBackend/ComfyUIBackendExtension.cs`

적용 내용:

- `CheckForUpdates`에서 `DLNodes` 폴더를 열거하기 전에 폴더 존재 여부를 확인한다.
- 폴더가 없으면 노드 업데이트 체크를 건너뛰고 정상 종료한다.

배경:

- `DLNodes`는 ComfyUI를 SwarmUI 내부에서 Self-Start 방식으로 설치할 때 생성된다.
- 외부 URL 방식(`ComfyUI API By URL`)으로 연결하면 이 폴더가 없으므로 `DirectoryNotFoundException`이 발생했다.

### 5. Ctrl+C 종료 문제 수정

수정 파일:

- `src/Core/Program.cs`

적용 내용:

- `Console.CancelKeyPress` 핸들러를 추가했다.
  - `e.Cancel = true`로 런타임 기본 강제종료를 막고 `Shutdown()`을 명시적으로 호출한다.
- Shutdown 시 Webhook 대기 타임아웃을 2분에서 5초로 단축했다.

배경:

- 기존에는 `CancelKeyPress` 핸들러가 없어서 Ctrl+C 후 .NET 런타임이 SIGINT를 처리하는 방식이 macOS에서 불안정했다.
- Webhook 대기가 최대 2분이라 종료 처리가 오래 걸리거나 프로세스가 멈추는 경우가 있었다.
