# 2026-04-10 업데이트 기록

## 이번 수정 목표

이번 수정의 목표는 아래 세 가지였다.

1. `Comfy Workflow`에서 현재 사용자 작업만 `작업 대기열`에 보이게 만든다.
2. `Comfy Workflow` 완료 결과를 상단 `Gallery`에 연결한다.
3. `Gallery`를 상단 탭으로 단순화하고, 다운로드 가능한 결과 목록처럼 보이게 만든다.

---

## 코드 변경

### 1. `Comfy Workflow` 작업 대기열 분리

수정 파일:

- `src/BuiltinExtensions/ComfyUIBackend/ComfyUIRedirectHelper.cs`
- `src/BuiltinExtensions/ComfyUIBackend/ComfyUser.cs`

적용 내용:

- `prompt_id` 소유권을 현재 사용자 기준으로 기록하도록 보강했다.
- `GET /ComfyBackendDirect/queue` 응답은 현재 사용자 소유 `prompt_id`만 남기도록 필터링했다.
- `GET /ComfyBackendDirect/history` 응답도 현재 사용자 소유 `prompt_id`만 남기도록 필터링했다.
- `POST /ComfyBackendDirect/queue`, `POST /ComfyBackendDirect/history` 삭제 요청은 현재 사용자 소유 `prompt_id`만 backend로 전달하도록 정리했다.
- websocket 이벤트도 `prompt_id` 기준으로 현재 사용자 작업만 통과시키도록 유지했다.

결과:

- `Comfy Workflow`의 `작업 대기열` 패널에서 실행 중 상태와 완료 이력이 현재 사용자 기준으로 보이게 맞췄다.

### 2. `Comfy Workflow` 결과를 상단 `Gallery`로 동기화

수정 파일:

- `src/BuiltinExtensions/ComfyUIBackend/ComfyUIRedirectHelper.cs`
- `src/Accounts/UserOutputHistoryIndex.cs`
- `src/Core/WebServer.cs`
- `src/WebAPI/T2IAPI.cs`

적용 내용:

- `Comfy Workflow` 완료 이력을 읽을 때 backend output 파일을 현재 사용자 output으로 저장하도록 연결했다.
- 저장 파일명은 `comfy-workflow-...` 패턴으로 고정했다.
- 결과 인덱스는 `LiteDB`에 저장하고, `OutputIndex/{entryId}` 라우트로 다운로드하게 했다.
- gallery/history 조회는 공유 폴더 전체 스캔이 아니라 인덱스 기반으로 바꿨다.

결과:

- backend `ComfyUI`에 저장된 결과를 현재 `SwarmUI` 인스턴스의 gallery용 인덱스로 끌어올 수 있게 만들었다.

### 3. 상단 `Gallery` 탭 정리

수정 파일:

- `src/Pages/Text2Image.cshtml`
- `src/Pages/_Generate/GalleryTab.cshtml`
- `src/Pages/_Generate/GenerateTab.cshtml`
- `src/wwwroot/js/genpage/gentab/outputhistory.js`
- `src/wwwroot/js/genpage/main.js`
- `src/wwwroot/js/genpage/gentab/currentimagehandler.js`

적용 내용:

- 하단 `History/Gallery` 구조를 제거하고 상단 top-level `Gallery` 탭을 추가했다.
- `Gallery`는 `Copy Path`, `Download`, `Download Metadata`, `Copy Raw Metadata` 중심으로 동작하게 정리했다.
- `Gallery`에서는 표시 형식, depth, 상위 폴더, 필터 같은 불필요한 기본 제어를 숨겼다.
- `View In Gallery` 동작을 현재 이미지 핸들러와 연결했다.

결과:

- 사용자는 상단 `Gallery` 탭에서 자기 결과물 목록을 바로 볼 수 있게 됐다.

### 4. macOS 실행 안정화

수정 파일:

- `src/Core/Program.cs`

적용 내용:

- macOS에서 포트 자동 변경 검사 중 실행이 멈추거나 예외가 나는 경로를 피하도록 수정했다.
- macOS에서는 설정 포트를 그대로 사용하도록 바꿨다.

결과:

- 로컬 macOS 검증 환경에서 최신 빌드를 다시 띄워 확인할 수 있게 됐다.

## 검증 기록

### 1차 검증: 빌드

실행:

- `dotnet build src/SwarmUI.csproj --configuration Release -o src/bin/live_release`

결과:

- 경고 0개
- 오류 0개

### 2차 검증: `Comfy Workflow` 실행 중 큐 표시

검증 방식:

- 실제 브라우저에서 `Comfy Workflow`를 열고 `app.queuePrompt(...)`로 실행
- `KSampler` 진행률과 `1개 활성`, `1개 실행 중` 문구 확인

확인 내용:

- `작업 대기열` 패널에 실행 중 상태가 표시됐다.
- 진행률(`총: xx%`, `KSampler : xx%`)과 완료 파일 목록이 보였다.

### 3차 검증: `Comfy Workflow` 완료 이력 표시

검증 방식:

- 같은 브라우저 세션에서 완료까지 기다린 뒤 `작업 대기열` 패널 확인

확인 내용:

- `0개 활성`
- `완료됨`
- 완료 파일명(`ComfyUI_00011_.png` 등) 표시

### 4차 검증: `Gallery` 반영

검증 방식:

- `Comfy Workflow` 결과를 인덱스 API와 output 파일에서 동시에 확인
- `Output/local/comfy-workflow-*.png` 저장 여부 확인
- 브라우저 `Gallery` 탭에서 결과 카드 표시 확인

확인 내용:

- `Output/local/comfy-workflow-2026-04-10-0305001-1.png`까지 실제 저장됨
- 상단 `Gallery` 탭에서 `comfy-workflow-...png` 카드와 다운로드 버튼이 표시됨

## 현재 남은 점검 포인트

1. 브라우저 캐시가 강하면 새 정적 JS가 즉시 반영되지 않을 수 있다.
2. 그래서 배포 시에는 `live_release` 재빌드 후 브라우저 강력 새로고침까지 같이 안내하는 편이 안전하다.
3. 현재 목표였던 `작업 대기열`, `완료 이력`, `상단 Gallery 표시`는 직접 실행으로 확인했다.
